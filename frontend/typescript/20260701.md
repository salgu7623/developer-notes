가상 돔
1. 상태가 바뀌면 컴포넌트가 리렌더링 되는데, 이 때 리렌더링 되더라도 실제 화면에서는 깜빡이면서 렌더링 되지 않는다. 왜그럴까?
그게 리액트의 가상돔 때문이다. 리액트는 현재 화면을 가상 돔에 그려두는데, 렌더링 시 가상돔과 diff 를 비교하여, 변경된 부분만 찾아서 실제 DOM 에 그려주기 때문에 다시 렌더링 되더라도 전체가 렌더링 되지 않는다.

Memo
1. 부모 컴포넌트의 상태가 변하면 다시 렌더링 되는데, 이 때 하위 자식 컴포넌트도 모두 렌더링 된다
2. 하위 자식 컴포넌트는 자신의 상태가 바뀐게 없는데 다시 렌더링되는 걸 피하고 싶다. => memo 로 wrapping
memo 로 wrapping 된 컴포넌트는 부모로부터 전달된 Props 가 바뀐게 있는지 비교하여 바뀐게 없으면, 부모가 렌더링 되더라도 다시 렌더링 되지 않는다.
즉, 부모 컴포넌트가 렌더링 됨 ➔ 원래라면 자식도 억울하게 같이 렌더링 되어야 함.
하지만 자식이 memo를 입고 있다면? ➔ "잠깐! 부모 렌더링 된 건 알겠는데, 부모가 나한테 준 데이터(props) 방금 전이랑 똑같아? 똑같으면 난 내 화면 안 바꾸고 예전 거 그대로 쓸래!" 하고 렌더링을 방어
3. 그럼 모든 컴포넌트를 다 memo 로 wrapping 하면 좋지 않나? X, memo 도 자신의 상태가 바뀐게 있는지 비교 연산이 들어가므로, 어차피 맨날 렌더링될 운명인 컴포넌트이거나 가벼운 컴포넌트들은 비교 연산이 추가되어 손해이므로 그냥 둔다.

[최종 정리]
1. 부모컴포넌트에 useState 로 관리되는 값이 변경되면, 부모컴포넌트가 리렌더링된다.

2. 부모컴포넌트가 리렌더링 되면, 당연히 아래에 있는 자식컴포넌트도 리렌더링된다.

3. 이 때, 자식컴포넌트는 부모로부터 전달받은 Props 가 바뀌지 않았는데도 리렌더링 되는게 억울할수있는데, 그래서 부모로부터 전달받은 Props 가 바뀌지 않았다면 자식컴포넌트는 리렌더링 되지 않도록 하는게 memo이다. memo 로 감싸진 컴포넌트는 부모컴포넌트의 상태가 바뀌더라도, 부모컴포넌트가 자신에게 넘겨주는 Props 가 바뀌지 않았다면 리렌더링 되지 않는 컴포넌트가 된다.

4. 이 때, memo 의 경우 얘기했듯이 그렇다면 부모컴포넌트가 자신에게 넘겨주는 Props 가 바뀌게 된다면 memo 로 감싸진 컴포넌트라 하더라도 리렌더링 되게 된다.

5. 자식컴포넌트가 만약에 부모컴포넌트로부터 함수를 props 로 받는다고 가정해보자. 그런 경우에 부모컴포넌트가 리렌더링 되면, 부모컴포넌트에 일반함수의 경우 새로운 메모리에 다시 생성이되면서 함수가 다시 만들어지므로, 그 함수를 props 로 받는 자식 컴포넌트도 리렌더링이 될 것이다.(memo 로 감싸있다할지라도). 

6.이를 막기 위해, useCallback 함수를 사용해서, useCallback(함수, []) 로 하면, 해당 함수는 callback 함수로사용될것이며, 이는 리렌더링 시에도 새로운 메모리에 할당이 되지 않고 기존 메모리에 그대로 남아있는 함수가 되고 해당 콜백함수를 Props 로 넘겨준다면, Props 가 변경되지 않기 때문에 자식 컴포넌트가 리렌더링 되지 않을 것이다

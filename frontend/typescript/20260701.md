가상 돔
1. 상태가 바뀌면 컴포넌트가 리렌더링 되는데, 이 때 리렌더링 되더라도 실제 화면에서는 깜빡이면서 렌더링 되지 않는다. 왜그럴까?
그게 리액트의 가상돔 때문이다. 리액트는 현재 화면을 가상 돔에 그려두는데, 렌더링 시 가상돔과 diff 를 비교하여, 변경된 부분만 찾아서 실제 DOM 에 그려주기 때문에 다시 렌더링 되더라도 전체가 렌더링 되지 않는다.

Memo
1. 부모 컴포넌트의 상태가 변하면 다시 렌더링 되는데, 이 때 하위 자식 컴포넌트도 모두 렌더링 된다
2. 하위 자식 컴포넌트는 자신의 상태가 바뀐게 없는데 다시 렌더링되는 걸 피하고 싶다. => memo 로 wrapping
memo 로 wrapping 된 컴포넌트는 부모로부터 전달된 Props 가 바뀐게 있는지 비교하여 바뀐게 없으면, 부모가 렌더링 되더라도 다시 렌더링 되지 않는다.
즉, 부모 컴포넌트가 렌더링 됨 ➔ 원래라면 자식도 억울하게 같이 렌더링 되어야 함.
하지만 자식이 memo를 입고 있다면? ➔ "잠깐! 부모 렌더링 된 건 알겠는데, 부모가 나한테 준 데이터(props) 방금 전이랑 똑같아? 똑같으면 난 내 화면 안 바꾸고 예전 거 그대로 쓸래!" 하고 렌더링을 방어
3. 그럼 모든 컴포넌트를 다 memo 로 wrapping 하면 좋지 않나? X, memo 도 자신의 상태가 바뀐게 있는지 비교 연산이 들어가므로, 어차피 맨날 렌더링될 운명인 컴포넌트이거나 가벼운 컴포넌트들은 비교 연산이 추가되어 손해이므로 그냥 둔다.
